Mathematica for Program Transformation

  • Overview

    Mathematica is one of the most advanced and comprehensive "mathematical software system", widely used by engineers, physicists and mathematicians. It is so attractive because of its comprehensive mathematical algorithms library. However, there is an distinguishing feature that has not been fully recognized.
    • The avaliability of built-in algorithms for manipulating symbolic expressions and, in particular, transformation rules, which makes it possible to write one's own transformation systems.
    • The possibility for defining one's own notation.
    The main goal of this project is to clarify this feature and implement a small transformation system, say parallelization.

  • Readings

    B. Buchberger. Mathematica as a rewrite language. Fuji'96.

    Z. Hu, M. Takeichi. A calculational framework for parallelization of sequential programs. IPSJ'97.

Extensible Java

  • Overview

    Java becomes hotter and hotter, not only in programming but also in implementation. Pizza is an example of such extension of Java with several new aspects of modern functional languages. It would be helpful if Java could provide an open mechanism for adding new language features.
    The aim of this work is to make Java be extensible by means of the refection technique. It would be pursuasive to show that pizza's new language features could be easily put into the entensibel Java.

  • Readings

    Some textbooks on Java

    Shigeru Chiba. A Metaobject Protocol for C++. OOPSLA'95.

    Martin Odersky. Pizza

Implementing Polytypic Functions in CRML

  • Overview

    CRML is a reflective ML language, developed by Tim Sheard in OGI. To what we know, this is the first reflective typed functional language. On the other hand, polytyism is a new abstraction features over data type. Note that CAML system has been installed in our lab.
    The goal of this work is to add the polytypic features to CRML by making full use of reflective mechanism provided by CRML. This will pave the way for further development of safe reflective system.

  • Readings

    CRML (Compile-time Reflective ML)

    P. Jansson, and J. Jeuring. PolyP - a polytypic programming language extension. In Principles of Programming Languages, pages 470--482, ACM Press, 1997.

    P. Jansson. Functional Polytypic Programming Use and Implementation, Computing Science, Chalmers University of Technology, 1997.
